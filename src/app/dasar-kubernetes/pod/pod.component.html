3. Pod

Pod
adalah suatu objek Kubernetes yang membungkus container yang berisikan aplikasi beserta konfigurasi yang menyertainya seperi volume, port, dll.

Pod membungkus containers (ya, bisa lebih dari satu container) untuk dijalankan dengan beberapa konfigurasi tambahan. Pod dapat menjadi satu objek yang berdiri sendiri, atau diatur oleh Pods Controller. Untuk melihat daftar pod yang sedang berjalan, dapat menggunakan perintah

kubectl get pods --all-namespaces

parameter --all-namepaces memerintahkan untuk menunjukkan semua pods yang ada. Jika, ingin melihat pods dari suatu namespace, ganti parameter menjadi --namespace <namespace_name>

Untuk menjalankan sebuah pod, secara sederhana dapat menggunakan perintah berikut

xxxxxxxxxx
kubectl run <pod_name> --image=<image_name>

Perlu diperhatikan bahwa container yang berjalan harus menjalankan command yang ada di dalam image secara terus-menerus atau pod akan mengalami CrashLoopBackOff dan memulai ulang kembali menjalankan container. Pasang parameter --restart=Never untuk mematikan pod setelah selesai menjalankan command dalam image

Untuk mengubah command dalam image, gunakan parameter --command -- <command> <args>.

Untuk membuat pod tetap berjalan tanpa harus mengganti command, gunakan parameter --attach=false -i -t. Ketiga parameter ini secara berurut adalah untuk, detach dari container, tetap menyalakan stdin dalam container, dan mengalokasikan terminal ke dalam container.

Untuk menghapus sebuah pod, gunakan perintah berikut

xxxxxxxxxx
kubectl delete pod <pod_name> [--namespace <namespace>]

Di dalam sebuah pod, setidaknya harus ada satu buah container. Pod juga bisa memiliki volume yang dapat di-mount ke container yang ada di dalam pod

Container
adalah sebuah pembungkus aplikasi yang terpisah dari sistem operasi yang ada di dalam mesin atau bahkan memiliki sistem operasi sendiri di dalamnya.

Container tidak bisa berdiri sendiri dalam cluster. Container harus dibungkus di dalam sebuah pod. Berikut adalah struktur dari sebuah container

name: <container_name: string>
image: <image_name: string>
imagePullPolicy: <image_pull_policy: string>
workingDir: <working_dir: string>
command: [<command: string>]
args: [<args: string>]
ports:
  - name: <port_name: string>
    containerPort: <port_number: string>
    protocol: <port_protocol: string>
env:
  - name: <env_name: string>
    value: <env_value: string>
  - name: <env_name: string>
    valueFrom:
      fieldRef:
        fieldPath: <field_ref: string>
  - name: <env_name: string>
    valueFrom:
      resourceFieldRef:
        resource: <resource_field_ref: string>
volumeMounts:
  - name: <volume_name: string>
    mountPath: <mount_path: string>
    subPath|subPathExpr: <sub_path: string>
    readOnly: <read_only: boolean>
resources:
  requests:
    cpu: <number_of_cpu>
    memory: <size_of_memory>
  limits:
    cpu: <number_of_cpu>
    memory: <size_of_memory>

Berikut adalah daftar nilai yang mungkin dari parameter env[].valueFrom.fieldRef.fieldPath

Berikut adalah daftar nilai yang mungkin dari parameter env[].valueFrom.resourceFieldRef.resource

Untuk struktur dari volumes, dapat dilihat pada bab 7. Storage subbab Volume

Berikut adalah contoh berkas yaml dari pod

xxxxxxxxxx
apiVersion: v1
kind: Pod
metadata:
  name: ubuntu
  namespace: default
  labels:
    app: ubuntu
spec:
  restartPolicy: Always
  containers:
    - name: ubuntu
      image: ubuntu:focal
      command: ["/bin/bash", "-c", "--"]
      args: ["tail -f /dev/null"]

Secara default, sebuah pod dapat diakses dari pod lain dalam domain <pod-ip>.<namespace>.pod.<cluster_domain> di mana <pod-ip> adalah IP dari pod dengan mengganti tanda . menjadi - (misal 10.244.3.10 menjadi 10-244-3-10) dan <cluster_domain> secara default adalah cluster.local. Sebagai contoh, pod dengan IP 10.244.3.10 dalam namespace default dapat diakses dengan 10-244-3-10.default.pod.cluster.local

Sebuah pod memiliki tiga klasifikasi Quality of Service berdasarkan field spec.containers[].resources dari pod tersebut, di antaranya:

  1. Guarantee, yaitu suatu pod yang mana setiap container di dalamnya diatur semua nilai parameter spec.resources sedemikian rupa sehingga spec.resources.limits.cpu sama dengan spec.resources.requests.cpu dan spec.resources.limits.memory sama dengan spec.resources.requests.memory.
  2. Burstable, yaitu suatu pod yang tidak memiliki kriteria seperti klasifikasi Guarantee tetapi memiliki setidaknya satu container yang mengatur nilai parameter spec.resources.requests.cpu atau spec.resources.requests.memory.
  3. BestEffort, yaitu suatu pod yang setiap container di dalamnya tidak mengatur nilai parameter spec.resources.
