3. Menggunakan berkas config Kubernetes

Secara default, pengguna tidak bisa secara langsung mengakses cluster melalui kubectl. Agar pengguna dapat mengakses cluster, salin berkas /etc/kubernetes/admin.conf di dalam mesin master node menjadi $HOME/.kube/config di dalam mesin yang ingin mengakses cluster. Aplikasi kubectl akan langsung mengenali berkas config di dalam $HOME/.kube. Pengguna dapat memeriksa apakah akses berhasil dengan mencoba perintah

xxxxxxxxxx
kubectl get nodes

Keluarannya kira-kira akan seperti ini

​x
NAME       STATUS   ROLES    AGE   VERSION
master-0   Ready    master   10m   v1.19.2
worker-1   Ready    <none>   10m   v1.19.2
worker-2   Ready    <none>   10m   v1.19.2
worker-3   Ready    <none>   10m   v1.19.2
​

Isi dari berkas config secara garis besar adalah seperti berikut

xxxxxxxxxx
apiVersion: v1
clusters:
- cluster:
    certificate-authority-data: <certificate_authority_data>
    server: https://<control_plane_ip>:<control_plane_port>
  name: <cluster_name>
- cluster:
    certificate-authority-data: <certificate_authority_data>
    server: https://<control_plane_ip>:<control_plane_port>
  name: kubernetes
contexts:
- context:
    cluster: <cluster_name>
    user: <user_name>
  name: <context_name>
- context:
    cluster: kubernetes
    user: kubernetes-user
  name: kubernetes-for-user
current-context: <context_name>
kind: Config
preferences: { }
users:
- name: <user_name>
  user:
    client-certificate-data: <client_certificate_data>
    client-key-data: <client_key_data>
- name: kubernetes-user
  user:
    token: <token>

Untuk melihat daftar context yang ada di dalam berkas config, jalankan perintah

xxxxxxxxxx
kubectl config get-contexts

Kira-kira keluarannya akan seperti berikut untuk contoh di atas

xxxxxxxxxx
CURRENT   NAME                   CLUSTER      AUTHINFO            NAMESPACE
*         kubernetes-for-admin   kubernetes   kubernetes-admin
          kubernetes-for-user    kubernetes   kubernetes-user

Untuk berganti context dapat digunakan perintah berikut

xxxxxxxxxx
kubectl config use-context <context_name>

Untuk menambah cluster ke dalam config, gunakan perintah

xxxxxxxxxx
kubectl config set-cluster <cluster_name> --server=https://<control_plane_ip>:<control_plane_port> --certificate-authority=<certificate_authority_data>

Untuk menambah user ke dalam config, gunakan perintah

xxxxxxxxxx
kubectl config set-credentials <user_name> --client-certificate=<client_certificate_data> --client-key=<client_key_data>

atau

xxxxxxxxxx
kubectl config set-credentials <user_name> --token=<token>

Sedangkan untuk membuat context, gunakan perintah

xxxxxxxxxx
kubectl config set-context <context_name> --cluster=<cluster_name> --user=<user_name>

Untuk menghapus context/cluster/user di dalam config, gunakan perintah

kubectl config unser [contexts|clusters|users].<object_name>
