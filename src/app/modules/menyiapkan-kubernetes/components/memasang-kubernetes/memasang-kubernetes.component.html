1. Memasang Kubernetes

Sebelum menyiapkan Kubernetes ke dalam sistem mesinmu, kamu harus memasang Docker di dalam setiap mesin yang ingin kamu masukkan ke dalam cluster Kubernetes. Untuk menyiapkan Kubernetes, dibutuhkan tiga packages, yaitu:

  1. kubeadm, bertugas untuk mengatur cluster dan hubungan antar mesin yang menjadi node dari cluster;
  2. kubelet, yang merupakan worker untuk melakukan perintah-perintah dari pengguna ke mesin seperti membuat container;
  3. kubectl, alat yang digunakan pengguna untuk berinteraksi dengan cluster.

Berikut akan dijelaskan bagaimana cara memasang ketiga packages di atas dalam mesin yang menjalankan sistem operasi Ubuntu 18.04.

Pertama, tambahkan repositori Kubernetes ke dalam daftar repositori mesin. Proses menambahkan repositori diawali dengan menambahkan signing key agar repositori yang ditambahkan bersifat autentik dan bukan repositori berbahaya

curl -s https://packages.cloud.google.com/apt/doc/apt-key.gpg | sudo apt-key add

Pastikan package curl terpasang dalam mesin

Kemudian, gunakan perintah ini untuk menambahkan repositori Kubernetes ke dalam daftar repositori

xxxxxxxxxx
sudo apt-add-repository "deb http://apt.kubernetes.io/ kubernetes-xenial main"

Setelah repositori ditambahkan, pasang tiga packages untuk Kubernetes dengan perintah

xxxxxxxxxx
sudo apt-get install kubeadm kubelet kubectl

Kemudian tandai ketiga packages tersebut agar tidak ter-update dan ter-autoremove dengan perintah

xxxxxxxxxx
sudo apt-mark hold kubeadm kubelet kubectl

Setelah semua mesin dipasang ketiga packages di atas, semua mesin yang akan dimasukkan ke dalam cluster harus mematikan swap memory dari mesin jika ada. Matikan dengan perintah berikut

xxxxxxxxxx
swapoff -a

Jangan lupa hapus baris yang merupakan informasi mengenai swap memory di dalam berkas /etc/fstab.

Dan jika mau, untuk membedakan tiap mesin, disarankan untuk menamai tiap mesin dengan nama unik menggunakan perintah

xxxxxxxxxx
hostnamectl set-hostname <nama_mesin>

Jangan lupa untuk menambahkan IP dan hostname dari semua mesin ke dalam berkas /etc/hosts sehingga tiap mesin bisa saling mengenali satu sama lain.
