4. Mendaftarkan private Docker registry

Jika pengguna memiliki Docker registry sendiri, pengguna dapat menambahkan registry tersebut ke dalam cluster sehingga cluster dapat mengakses registry tersebut. Ada dua metodenya

  1. Melakukan login melalui Docker dan mengirim Docker config ke cluster;
  2. Langsung mengirim parameter yang dibutuhkan untuk mengakses registry ke cluster.

Untuk metode pertama, pengguna harus login ke registry melalui Docker menggunakan perintah

docker login <server_registry> -u <username> -p <password>

Kemudian, pengguna mengirim Docker config yang biasanya ada di $HOME/.docker/config.json ke cluster dengan perintah

xxxxxxxxxx
kubectl create secret generic <secret_name> --from-file=.dockerconfigjson=<docker_config_file> --type=kubernetes.io/dockerconfigjson

Untuk metode kedua, pengguna bisa langsung menggunakan perintah

kubectl create secret docker-registry <secret_name> --docker-server=<server_registry> --docker-username=<username> --docker-password=<password> --docker-email=<email>

Kedua metode di atas sama-sama membuat object dari Kubernetes yaitu secret yang akan dijelaskan nanti pada bagian ini
