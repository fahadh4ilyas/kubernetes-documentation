2. Namespace

Namespace
adalah suatu objek dari Kubernetes yang berguna untuk mengelompokkan objek-objek dari Kubernetes.

Salah satu kegunaan namespace adalah pengguna dapat mengelompokkan objek-objek Kubernetes yang memiliki kesamaan dalam penggunaan. Namespace juga dapat digunakan untuk mengatur batasan penggunaan objek Kubernetes maupun resources dari mesin seperti CPU dan memory di dalamnya.

Secara default, dalam setiap cluster pasti terdapat namespace dengan nama default. Namespace default adalah namespace yang digunakan jika pengguna tidak secara eksplisit memilih namespace saat membuat objek Kubernetes.

Tapi, tidak semua objek dari Kubernetes perlu masuk dalam namespace. Beberapa objek yang tidak perlu masuk namespace adalah seperti ClusterRole, ClusterRoleBinding, PersistentVolume, dll.

Untuk mendapatkan daftar namespace yang ada, dapat menggunakan perintah

xxxxxxxxxx
kubectl get namespaces

keluarannya kira-kira akan seperti ini

xxxxxxxxxx
NAME              STATUS   AGE
default           Active   1h
kube-node-lease   Active   1h
kube-public       Active   1h
kube-system       Active   1h
lens-metrics      Active   1h

Untuk membuat namespace, dapat menggunakan perintah

xxxxxxxxxx
kubectl create namespace <namespace_name>

atau menggunakan berkas yaml seperti berikut

xxxxxxxxxx
apiVersion: v1
kind: Namespace
metadata:
  name: <namespace_name: string>

Untuk menghapus namespace, dapat menggunakan perintah

kubectl delete namespace <namespace_name>
